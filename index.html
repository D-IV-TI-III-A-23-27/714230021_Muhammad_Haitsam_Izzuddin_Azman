01 — Pengenalan

Apa itu Gradient Boosting?

💡

Penjelasan Sederhana

Bayangkan sebuah tim kerja yang sedang memecahkan masalah sulit. Anggota pertama mencoba menyelesaikan, tapi hasilnya masih belum sempurna dan ada bagian yang salah.

Anggota kedua tidak memulai dari nol, ia hanya fokus memperbaiki kesalahan yang dibuat anggota pertama. Lalu anggota ketiga masuk khusus untuk memperbaiki sisa kesalahan anggota kedua, begitu seterusnya.

Intinya: Gradient Boosting bekerja dengan menyatukan banyak model prediksi sederhana secara berurutan. Setiap model baru yang ditambahkan memiliki satu tugas utama: memperbaiki error (kesalahan) dari model sebelumnya. Kerja sama tim inilah yang menghasilkan prediksi yang sangat akurat!

Ensemble Method

Menggabungkan banyak model lemah (weak learner) menjadi satu model kuat (strong learner) dengan cara aditif dan sekuensial.

Gradient Descent

Menggunakan teknik gradient descent pada ruang fungsi untuk meminimalkan loss function secara iteratif di setiap tahap.

Decision Trees

Umumnya menggunakan decision tree sebagai base learner, yang dikenal sebagai Gradient Boosted Decision Trees (GBDT).

Minimasi Error

Setiap iterasi fokus pada residual (sisa error) dari prediksi sebelumnya, secara bertahap mengurangi total kesalahan.

03 — Proses

Langkah Kerja Gradient Boosting

1

Inisialisasi Model Awal

Model dimulai dengan prediksi konstan, biasanya rata-rata dari target variable. Ini menjadi base prediction F₀(x).
✨ Sederhananya: Membuat tebakan awal yang paling aman.

F₀(x) = argminγ Σ L(yᵢ, γ)
2

Hitung Pseudo-Residuals

Hitung gradient negatif dari loss function terhadap prediksi saat ini. Residuals ini menunjukkan arah perbaikan yang diperlukan.
✨ Sederhananya: Mencari tahu seberapa meleset tebakan sebelumnya.

rᵢₘ = -[∂L(yᵢ, F(xᵢ))/∂F(xᵢ)]F=Fₘ₋₁
3

Fit Weak Learner (Tree)

Latih decision tree baru pada pseudo-residuals. Tree ini belajar memprediksi koreksi yang diperlukan untuk memperbaiki prediksi saat ini.
✨ Sederhananya: Menugaskan anggota baru khusus untuk memperbaiki kesalahan tadi.

hₘ(x) = DecisionTree(X, rₘ)
4

Tentukan Optimal Step Size

Cari learning rate γ yang optimal untuk setiap region di leaf node, meminimalkan total loss function.
✨ Sederhananya: Menentukan seberapa besar langkah perbaikan yang akan diambil.

γₘ = argminγ Σ L(yᵢ, Fₘ₋₁(xᵢ) + γ·hₘ(xᵢ))
5

Update Model

Tambahkan weak learner yang telah di-scale dengan learning rate ke model ensemble. Proses ini diulang selama M iterasi.
✨ Sederhananya: Menggabungkan hasil perbaikan ke dalam sistem kerja tim.

Fₘ(x) = Fₘ₋₁(x) + ν · γₘ · hₘ(x)
6

Output Prediksi Final

Model akhir adalah jumlah dari semua weak learners. Prediksi dibuat dengan menjumlahkan kontribusi setiap tree.
✨ Sederhananya: Hasil akhir super akurat dari kerja sama seluruh anggota tim.

F(x) = F₀(x) + Σₘ₌₁ᴹ ν · γₘ · hₘ(x)

06 — Varian

Varian Populer Gradient Boosting

⚡

XGBoost

eXtreme Gradient Boosting — Implementasi yang sangat teroptimasi dengan regularisasi L1 & L2, parallel computing, dan penanganan missing values otomatis.

  • Regularisasi bawaan
  • Parallel tree construction
  • Cross-validation bawaan
🍃

LightGBM

Light Gradient Boosting Machine oleh Microsoft — Menggunakan teknik Gradient-based One-Side Sampling (GOSS) dan Exclusive Feature Bundling (EFB).

  • Leaf-wise tree growth
  • Sangat cepat dan efisien memori
  • Mendukung categorical features
🐱

CatBoost

Categorical Boosting oleh Yandex — Dirancang khusus untuk menangani fitur kategoris tanpa perlu encoding manual.

  • Ordered boosting
  • Penanganan kategori otomatis
  • Minim overfitting
